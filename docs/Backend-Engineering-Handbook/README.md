# Backend Engineering Handbook

## Purpose

This handbook documents my journey learning backend engineering with C# and ASP.NET Core.

Instead of memorizing syntax, I focus on understanding:

- What problem each concept solves
- Why it exists
- How it fits into the architecture
- How it compares to technologies I already know
- The mental models that made it click

Every chapter follows the same structure:

- What Problem Does This Solve?
- Solution
- Why This Matters
- Mental Model
- Real-World Example
- Code Example
- Common Beginner Questions
- Common Mistakes
- Interview Answer
- One-Sentence Summary
- What Finally Made It Click

The goal is to build a reference manual that explains *why* backend technologies exist, not just *how* to use them.
